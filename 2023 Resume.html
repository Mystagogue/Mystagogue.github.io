My Work

Puppet Master is a desktop app built with Electron and React. Electron allows you to use html, Css and JS to build desktop applications. The render process in an electron app is basically identical to the front end of a web app. The application allows you to create new instances of Chrome in context isolation from each other and it has a code editor that lets you write and execute puppeteer automation code on any of the new browser instances you have created. If, like me, you spned your days constantly checking out different branches of the same code, signing in with a multitude of different users with different permissions in order see the adjustments you are making to the code on the nth step of a multi step form then this app is for you! It means you never need to worry about cache or cookies, you always get the right users creds, it gets you there instantly. Go to the site and try it out!

This is a good showcase of my work in React. It involves some complex state management where the file tree in the application must mirror the file system of the OS and manage the state of currently open documents. Try to save a document that has changed in the file system since you opened it? It will give you a warning. Drag and drop a folder into a different location in the file tree? It will look at all open files and reassign their new path in state. That sort of thing. This app supports mutliple themes and when you change the theme of the code editor it changes the theme of the whole app to correspond. This is done with careful use of CSS variables.

In addition this app involves multi-processing and virtual machines. Whenever you start a new browser instance a browser manager is created which is a child process forked from main. Whenever you execute user input code it creates a virtual machine with a curated context to execute the code. Part of the context involves changing the require method so it will look in the currently open folder rather than inside the app itself.
